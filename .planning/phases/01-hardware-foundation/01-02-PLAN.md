<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| MCU <-> I2C1 Bus | Extended to 6 devices (5x INA226 + 1x DAC8571); any slave can fault |
| I2C1 Bus <-> DAC8571 | DAC output drives analog control loop — incorrect output could cause hardware damage |
| I2C1 Bus <-> INA226[1..4] | Alert pins connected to EXTI — but not enabled in Phase 1 (deferred to Phase 2) |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-02-01 | Spoofing | I2C address aliasing | mitigate | Each INA226_Dev.address explicitly set in array; no address scanning at runtime — wrong address = I2C_NACK at init, caught immediately |
| T-02-02 | Tampering | DAC8571 output overrange | mitigate | dac8571_set_output bounds: value is uint16_t (0-65535) which maps to 0-VREF — inherently bounded by DAC resolution. No software bounds check needed but value type constrains range |
| T-02-03 | Denial of Service | Multi-device I2C read loop timeout | mitigate | Each device read is independently timeout-protected via i2c_util; a stuck device at address N does not prevent reading device N+1. The reading loop continues after a timeout return. |
| T-02-04 | Denial of Service | DAC8571 not present | mitigate | dac8571_init() verifies ACK with probe write; if DAC is absent, init prints FAIL and loop skips DAC operations — other devices continue functioning |
| T-02-05 | Information Disclosure | INA226 calibration values in debug output | accept | Debug output is development-only (USART1 PA9 pin); no calibration secrets exposed (all values are #define constants, public knowledge from datasheet) |
| T-02-06 | Tampering | INA226 alert register misread | monitor | check_alert() reads and returns raw mask; application in Phase 2 will interpret and act. In Phase 1 the value is informational only — no safety action is taken based on alert flags |
| T-02-SC | Tampering | Package installs | accept | No external packages — all from SPL V2.9. N/A per RESEARCH.md Package Legitimacy Audit. |
</threat_model>
