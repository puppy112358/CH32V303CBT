# Plan 02: cJSON Integration + Command Parsing + Dispatch

## Objective

Vendor the cJSON library (v1.7.18) into `Drivers/cjson/`, configure a 4 KB static arena allocator via `cJSON_InitHooks()`, and implement the complete command parsing and dispatch pipeline in `protocol.c`. Support all four commands: `set_mode` (CV/CC with engineering-unit values), `clear_fault` (FAULT→IDLE transition), `get_status` (runtime state), and `get_info` (static system info). Every command receives a JSON acknowledgment — `{"ack":"ok"}` on success, `{"ack":"error","code":N,"msg":"description"}` on failure. Full input validation: JSON syntax, required fields, value ranges, mode strings, state gating. Wire `protocol_poll()` into the main loop at the start of each 100ms control cycle. This delivers COMM-01 (UART0 cJSON command control).

## must_haves

- cJSON v1.7.18 vendored as `Drivers/cjson/cJSON.h` + `Drivers/cjson/cJSON.c` (single-header/source library)
- Static arena allocator: 4 KB `cjson_pool[]` with bump-allocator hooks set via `cJSON_InitHooks()`
- Arena reset after each parse cycle: `cjson_pool_used = 0` reclaims all memory
- `protocol_process_command()`: parse → validate → dispatch → respond → cleanup
- Four commands: `set_mode` (CV/CC), `clear_fault`, `get_status`, `get_info`
- All responses use `cJSON_PrintUnformatted()` — compact, single-line, machine-readable
- Error responses with numeric codes in 1xx/2xx/3xx ranges and human-readable `msg` strings
- Input validation: valid cJSON syntax, required fields (`cmd` always; `mode`/`value` for set_mode), value ranges (CV: 0.0–30.0V, CC: 0.0–10.0A), valid mode strings ("CV"/"CC" only), known commands
- State gating per D-14: `set_mode` rejected in FAULT; `clear_fault` rejected outside FAULT; same-mode same-value is idempotent
- `clear_fault` calls `fault_clear()` (already declared in fault.h) — fills in the TODO stub
- `protocol_poll()` called at the start of each control cycle in main.c, before sensor reads
- cJSON arena reset after each complete command cycle (parse → dispatch → respond → free)

## truths

- cJSON uses `cJSON_malloc`/`cJSON_free` function pointers — overriding them with arena hooks prevents heap fragmentation
- The arena is a bump allocator: `cjson_arena_alloc()` returns `&cjson_pool[used]`, advances `used += sz`; `cjson_arena_free()` is a no-op
- Commands are processed BEFORE telemetry assembly (Plan 03) — no concurrent cJSON trees, 4 KB pool is sufficient
- Typical command parse tree: 50–100 bytes JSON → ~1–2 KB cJSON tree (each node ~56 bytes)
- `cJSON_PrintUnformatted()` produces compact JSON without whitespace — saves ~20–50 bytes per response vs `cJSON_Print()`
- `fault_clear()` stub in fault.c (line 118 of fault.h) needs implementation: transition FAULT→IDLE, zero fault_reg
- `engage_cv()` and `engage_cc()` already exist in main.c — command dispatch calls them directly
- `system_mode`, `cv_target_voltage`, `cc_target_current` are globals in main.c — command dispatch reads/writes them
- `DBGMCU_GetCHIPID()` returns `uint32_t` — format as 8-char hex for get_info
- System constants: `#define HW_MAX_VOLTAGE 30.0f`, `#define HW_MAX_CURRENT 10.0f`, `#define FW_VERSION "1.0.0"`

      - Send ack: `{"ack":"ok"}`

   b. `strcmp(cmd, "clear_fault") == 0`:
      - State gate: if system_mode != MODE_FAULT → error code=302 "Not in fault state"

      - Send ack: `{"ack":"ok"}`

   c. `strcmp(cmd, "get_status") == 0`:

## Threat Model

| Threat | Severity | Mitigation |
|--------|----------|------------|
| cJSON parse of untrusted input causes buffer overflow | High | cJSON v1.7.18 is fuzz-tested; arena allocator bounds-checks all allocations (returns NULL at 4 KB limit); all string copies use cJSON's internal length-checked routines |
| Arena pool exhaustion from large/malicious JSON | Medium | 4 KB pool covers ~20× typical command size; NULL return from arena triggers graceful error ack 101; pool reset each cycle prevents accumulation |
| Command injection via crafted mode/value strings | Medium | strcmp exact match on mode strings ("CV"/"CC" only); value range gated against HW_MAX constants; numeric type checked via cJSON_GetObjectItem before use |
| Fault state bypass via rapid set_mode while fault_triggered | Low | Command processing happens BEFORE the fault check in the control cycle (step 0 before step 3); if fault_triggered was set by ISR between cycles, it is checked at step 3 and takes priority; worst case: one cycle delay |
| clear_fault race with auto-retry in fault_state_machine | Low | clear_fault sets fault_reg.bits.latched=0 and system_mode=MODE_IDLE atomically (within the same cycle, before fault_state_machine runs); fault_state_machine only acts when system_mode==MODE_FAULT |
| cJSON_PrintUnformatted allocation failure | Low | Output strings are small (<200 bytes for responses, <350 bytes for telemetry); arena pool reset before assembly ensures full 4 KB available; NULL return handled |
</attack_vectors>
