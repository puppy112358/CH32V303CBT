# Plan 03: Telemetry Assembly + TX + Main Loop Integration

## Objective

Implement `protocol_send_telemetry()` to assemble and transmit a cJSON telemetry packet each 100ms control cycle via USART2. The packet contains: 4-channel MOS voltage/current in a `ch[]` array, summary voltage/current/power in a `sum{}` object, and flat meta fields (sequence number, uptime in ms, active mode string, fault code, DAC value, retry count, temperature placeholder). Wire the telemetry call into the main loop after the state machine dispatch (after fault checks + PID compute, before the debug printf summary). This delivers COMM-02 (10Hz cJSON telemetry reporting) and completes all Phase 3 requirements.

## must_haves

- `protocol_send_telemetry()` assembles the full cJSON telemetry tree per D-07: `ch[]` array with `v`/`i` per channel, `sum{}` object with `v`/`i`/`p`, flat fields `seq`/`uptime`/`mode`/`fault`/`dac`/`retry`/`temp`
- Sequence number (`seq`): `uint16_t` increments each packet, rolls over at 65535
- Uptime (`uptime`): `uint32_t` milliseconds computed from `cycle_count × CONTROL_PERIOD_MS`
- Temperature (`temp`): `0.0` placeholder per D-11 — consistent schema from day one
- Packet transmitted via `protocol_send()` (blocking TX, ~26ms for ~300 bytes at 115200 bps)
- cJSON arena reset after telemetry assembly (reclaim all memory for next cycle)
- Telemetry called once per control cycle, after PID compute and state machine dispatch, before debug printf
- Debug printf (step 9 in main.c) simplified: either removed or reduced to minimal one-line summary since telemetry now provides full data

## truths

- Telemetry is synchronized with the control cycle per D-08: assembled and transmitted after PID update + fault state machine
- cJSON arena is reset before telemetry assembly (Plan 02 resets it after command processing; Plan 03 resets it after telemetry TX)
- The telemetry tree uses ~4–6 KB during assembly (cJSON structs + output string). The 4 KB arena is sufficient because telemetry is assembled AFTER command processing (no concurrent trees)
- Blocking TX at 115200 bps: 300 bytes × 11 bits/byte ÷ 115200 = ~28.6ms theoretical, ~26ms measured. Well within the 100ms control budget. Remaining ~70ms for sensor reads + PID + fault checks.
- `seq` rollover at 65535 → ~109 minutes at 10Hz before rollover. PC application should handle wrap.
- `uptime` in milliseconds: `uint32_t` max value = 4,294,967,295 ms ≈ 49.7 days before rollover
- MOS channel voltage: `ina226_get_bus_voltage()` returns float in volts; already called in main loop steps 1–2
- MOS channel current: `ina226_get_current()` returns float in amps; already called in main loop
- Summary voltage/current/power: `bus_v`, `bus_i`, `bus_p` floats from main loop step 1
- The INA226 readings for telemetry are the same values used for PID compute — no additional I2C reads needed
- Debug printf can remain as-is for development visibility, or be simplified. Phase 3 makes it optional since telemetry carries the same data.

## Threat Model

| Threat | Severity | Mitigation |
|--------|----------|------------|
| Telemetry TX blocks too long, misses control deadline | Medium | 300-byte packet at 115200 bps = ~26ms. Remaining cycle time ~74ms covers sensor reads (~15ms I2C) + PID (~1ms) + fault checks (~1ms). If telemetry grows >500 bytes, consider interrupt-driven TX. |
| cJSON arena exhaustion during telemetry assembly | Low | Arena reset at function entry ensures full 4 KB available. 300-byte JSON requires ~4-5 KB tree — may need arena increase to 5 KB. Monitor via cjson_pool_used audit printf (debug only). |
| Sequence number race across cycles | Very Low | telemetry_seq is a static local — only protocol_send_telemetry() modifies it, called once per cycle from the main loop. No concurrent access. |
| Uptime drift from SysTick inaccuracy | Very Low | SysTick at 96 MHz with 8 MHz HSE crystal — ppm-level accuracy. Drift <1 second per day. Uptime derived from fixed CONTROL_PERIOD_MS × cycle_count — deterministic, not SysTick dependent. |
| JSON output contains control characters breaking framing | Low | cJSON_PrintUnformatted escapes all strings per JSON spec. Newline only appears as the framing delimiter added by protocol_send(). |
| Telemetry leaks sensitive system state | Low | All telemetry fields are operational metrics — no secrets, keys, or credentials. Appropriate for unencrypted UART link in a lab/test environment. |
</attack_vectors>
