# Plan 02: NTC Temperature Sensor — ADC1 + Lookup Table + Telemetry Integration

## Objective

Create the `Drivers/temp_sensor.c/.h` module that reads the NTC thermistor (B=3950, 10kΩ at 25°C) via ADC1 on PA5 using a voltage divider (10kΩ fixed resistor to VDD). Pre-compute a 100-entry temperature lookup table mapping ADC readings to 0.1°C units. Provide `temp_sensor_read()` returning float °C. Replace the telemetry `temp` field placeholder `0.0` in `protocol_send_telemetry()` with the real temperature reading. This populates the telemetry temp field and provides the temperature feedback source for Plan 03 (fan PID).

## must_haves

- `Drivers/temp_sensor.h` with include guard `__DRIVERS_TEMP_SENSOR_H`, C++ extern block, public API: `temp_sensor_init()`, `temp_sensor_read()`
- `Drivers/temp_sensor.c` with ADC1 single-channel init (PA5, ADC_Channel_5, 239.5-cycle sample time), 100-entry `const int16_t ntc_temp_table[100]` lookup table, and `temp_sensor_read()` returning float °C
- ADC1 configured for single conversion mode (software trigger, no continuous conversion, no DMA, no interrupt) — called synchronously from main loop
- NTC lookup table pre-computed from B-parameter equation: `R_ntc = 10000 * (4095.0 / ADC - 1.0)` and `T_C = 1.0 / (ln(R_ntc/10000.0) / 3950.0 + 1.0/298.15) - 273.15`
- Table range: ADC 50–4000 covering ~0°C to ~100°C at ~1°C resolution. Table stored as int16_t in 0.1°C units with +40°C offset (allows -40°C to 100°C range).
- `temp_sensor_read()` returns `(heatsink_temp_01c - 400) / 10.0f` converting from 0.1°C fixed-point to float °C
- `protocol_send_telemetry()` in protocol.c: line 574 `cJSON_AddNumberToObject(root, "temp", 0.0)` changed to read the global `heatsink_temp_c` via extern
- `temp_sensor_init()` called from main.c init section after WS2812 init, before super-loop
- `temp_sensor_read()` called from main.c super-loop each 100ms cycle (new step 5.5: NTC read)
- Follows existing Drivers/ module conventions

## truths

- PA5 = ADC12_IN5 = ADC1 channel 5 — confirmed free (PA0=TIM2 CH1, PA2/PA3=USART2, PA4=EXTI4, PA9=USART1, PA11/PA12=USB)
- Voltage divider: fixed 10kΩ from PA5 to VDD (3.3V), NTC from PA5 to GND. At 25°C: R_ntc=10kΩ, V_ntc=1.65V, ADC≈2047. As temp increases: R_ntc decreases, V_ntc decreases, ADC decreases. As temp decreases: R_ntc increases, V_ntc increases, ADC increases.
- ADC1 clock = APB2 (PCLK2) at 72MHz with ADC prescaler /6 → ADC clock = 12MHz (max for CH32V303)
- ADC sample time 239.5 cycles at 12MHz = ~20μs sample time. Single conversion total: ~30μs (sample + 12.5 conversion cycles). Well within 100ms budget.
- No ADC interrupt needed: single conversion is blocking but fast (~30μs). Called synchronously from main loop.
- New global variable: `volatile float heatsink_temp_c = 25.0f;` — declared in temp_sensor.c, extern'd by main.c and protocol.c. Initial value 25°C (room temp assumption) until first ADC read.
- NTC B-parameter equation: R(T) = R25 × exp(B × (1/(T+273.15) − 1/298.15)). Inverse: T = 1 / (ln(R/R25)/B + 1/298.15) − 273.15.
- Lookup table strategy: for each ADC value from ADC_MIN=50 to ADC_MAX=4000 in steps of NTC_ADC_STEP=40, compute T_C, convert to 0.1°C + 40°C offset, store as int16_t. Runtime: `idx = (adc - 50) / 40; temp_01c = ntc_temp_table[idx]; T_C = (temp_01c - 400) / 10.0f;`
- No runtime math.h calls — all exp()/log() done at compile time in table generation
- ADC reference = VDD (3.3V nominal). Accuracy depends on VDD stability. For v1, assume ±5% tolerance acceptable (fan control is forgiving).

4. Global variable: `volatile float heatsink_temp_c = 25.0f;` — default room temp until first ADC read

5. Lookup table constants:
   - `#define NTC_TABLE_SIZE 100`
   - `#define NTC_ADC_MIN 50` — minimum valid ADC reading (~100°C, R_ntc≈0Ω)
   - `#define NTC_ADC_MAX 4000` — maximum valid ADC reading (~0°C, R_ntc very high)
   - `#define NTC_ADC_STEP 40` — (4000-50)/100 ≈ 39.5 → use 40 for clean indexing
   - `#define NTC_TEMP_OFFSET 400` — +40°C offset in 0.1°C units (so 0 = -40°C)

6. Pre-computed lookup table `const int16_t ntc_temp_table[100]`:
   - This table must be HAND-COMPUTED and statically initialized (no runtime math).
   - For each index i = 0..99, ADC value = 50 + i*40:
     - R_ntc = 10000.0 * (4095.0 / ADC - 1.0)
     - T_K = 1.0 / (log(R_ntc / 10000.0) / 3950.0 + 1.0 / 298.15)
     - T_C = T_K - 273.15
     - temp_01c = (int16_t)((T_C + 40.0) * 10.0)
   - Key reference points (MUST match table):
     - ADC≈2047 at 25°C → temp_01c = (25.0 + 40.0) * 10 = 650
     - ADC≈1215 at 50°C → temp_01c = (50.0 + 40.0) * 10 = 900
     - ADC≈3286 at 0°C → temp_01c = (0.0 + 40.0) * 10 = 400
   - The actual table must be pre-computed offline. The .c file contains a comment showing the Python/B-equation used, followed by the 100-element int16_t array literal.
   - NOTE: The exact values should be pre-computed by the executor using the B-parameter equation. Include the formula as a comment above the array for reproducibility.

7. Implement `temp_sensor_read()`:
   - Trigger ADC conversion: `ADC_SoftwareStartConvCmd(ADC1, ENABLE);`
   - Wait for conversion complete: `while(ADC_GetFlagStatus(ADC1, ADC_FLAG_EOC) == RESET);`
   - Read result: `uint16_t adc = ADC_GetConversionValue(ADC1);`
   - Clamp: if adc <= NTC_ADC_MIN, return 100.0f (max temp, short circuit). If adc >= NTC_ADC_MAX, return -40.0f (min temp, open circuit).
   - Compute index: `uint8_t idx = (adc - NTC_ADC_MIN) / NTC_ADC_STEP;` — integer division
   - If idx >= NTC_TABLE_SIZE: idx = NTC_TABLE_SIZE - 1;
   - Lookup: `int16_t temp_01c = ntc_temp_table[idx];`
   - Convert to °C: `float temp_c = (float)(temp_01c - NTC_TEMP_OFFSET) / 10.0f;`
   - Update global: `heatsink_temp_c = temp_c;`
   - Return temp_c

All local variables declared at top of function (C89 convention).
</action>

## Threat Model

| Threat | Severity | Mitigation |
|--------|----------|------------|
| NTC open circuit (broken wire) → ADC reads ~4095 → temperature reports -40°C | Low | Temperature clamped to -40°C minimum. Fan PID sees very low temp → fan stays OFF. This is safe (no cooling needed at -40°C). Flag for v2: add sensor fault detection. |
| NTC short circuit → ADC reads ~0 → temperature reports 100°C | Low | Temperature clamped to 100°C maximum. Fan PID sees high temp → fan runs at 100%. This is safe (fails to max cooling). May cause unnecessary fan noise but no hardware damage. |
| ADC conversion hangs (EOC flag never set) → main loop stuck in while(EOC) poll | Medium | Mitigation: ADC EOC should always complete within ~30μs at 12MHz ADC clock. If EOC never fires, it indicates ADC clock failure or hardware fault — a rare but blocking scenario. For v1, accept this risk (same pattern as existing blocking I2C calls). v2 could add a tick-count timeout. |
| VDD voltage variation → ADC reference drift → temperature reading error | Low | VDD = 3.3V nominal. CH32V303 has internal voltage regulator. ±5% VDD variation → ±2-3°C temperature error. Acceptable for fan control (target is 50°C, fan activation is threshold-based, not precision measurement). |
| Lookup table entry calculation error → wrong temperature | Medium | Mitigation: table is statically initialized. Key reference points (0°C, 25°C, 50°C, 100°C) are verified in acceptance criteria. Range checks at table boundaries prevent out-of-bounds access. |
| No temperature reading on first telemetry cycle (heatsink_temp_c still 25.0f default) | Low | Initial value of 25°C is a safe default. First ADC read completes within ~30μs of temp_sensor_init() — well before first telemetry cycle (~100ms later). |
